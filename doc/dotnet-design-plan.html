Go → .NET Primitive Mapping

Every Go concurrency primitive has a direct .NET 8 equivalent — all in the BCL, zero NuGet. The goal is structural parity: same invariants, same data flows, different syntax and runtime model.

Concept Go Primitive .NET 8 Equivalent Notes
Lightweight concurrency goroutine Task + async/await ThreadPool-backed. Each conn gets a Task.Run read loop and a dedicated write loop.
Typed message channel chan T Channel<T> System.Threading.Channels. Bounded with DropOldest for QoS 0. BoundedChannelOptions replaces Go's select default.
Read-many / write-exclusive lock sync.RWMutex ReaderWriterLockSlim EnterReadLock / EnterWriteLock. Not async-safe — wrap async boundaries with SemaphoreSlim if needed.
Simple mutex sync.Mutex lock / SemaphoreSlim(1,1) Use lock for sync code. SemaphoreSlim(1,1) for async-safe critical sections with WaitAsync().
Concurrent key-value map sync.Map / map+mutex ConcurrentDictionary<K,V> Or sharded Dictionary + ReaderWriterLockSlim[] for the StateStore hot path.
Cancellation / context passing context.Context CancellationToken Propagate via method parameters. CancellationTokenSource.CreateLinkedTokenSource for hierarchy.
Condition variable sync.Cond SemaphoreSlim + Release Writers call WaitAsync(); flush loop calls Release(N) after fsync. More explicit than Cond.Broadcast.
Periodic background timer time.NewTicker PeriodicTimer .NET 6+ PeriodicTimer. Single-fire safe: await timer.WaitForNextTickAsync(ct). No drift accumulation.
Zero-copy network reads bufio.Reader System.IO.Pipelines.PipeReader True zero-copy. Exposes ReadAsync → ReadResult → SequenceReader<byte>. Built into .NET, no NuGet.
JSON encode/decode encoding/json System.Text.Json Use JsonSerializerContext + source generators for AOT compatibility. No reflection at runtime.
File append + sync os.File + Sync() FileStream(FileOptions.WriteThrough) Or RandomAccess.WriteAsync + fileStream.Flush(flushToDisk: true). Equivalent to fsync.
HTTP + WebSocket upgrade net/http + nhooyr.io/websocket HttpListener + AcceptWebSocketAsync Zero NuGet. HttpListener is BCL. AcceptWebSocketAsync(null) handles the HTTP→WS upgrade.
Unique ID generation github.com/oklog/ulid Guid.NewGuid() / UInt128 Guid.NewGuid().ToString("N") for msg_id. Or implement ULID in ~20 lines using System.Security.Cryptography.
Key Difference: Async Permeation Unlike Go where goroutines are cheap enough to use synchronously, .NET's async model permeates the entire I/O stack. The rule: any method that touches network, disk, or a Channel must be async Task. The ReaderWriterLockSlim used for shards is synchronous only — hold it for the minimum time, never across an await. For async-safe mutual exclusion, use SemaphoreSlim(1,1) instead.

Component Map

Seven components, identical structure to the Go design. Each maps directly to a .NET BCL type as its synchronization primitive. Hot path components are highlighted.

C1
Transport
HttpListener + System.Net.WebSockets
Accepts WebSocket upgrades. Each connection spins two Tasks: a PipeReader read loop and a Channel-drained write loop. Zero NuGet — BCL only.
C2
Message Router
ReaderWriterLockSlim + trie
Topic trie protected by RWLS. Read lock for Match(); write lock for Subscribe/Unsubscribe. Fan-out posts frames to per-session Channel<Frame>.
C3

ReaderWriterLockSlim[64]
64 shards keyed by topic.GetHashCode() & 63. Each shard is a Dictionary<string, JsonElement> guarded by its own RWLS. WAL write before memory update.
C4

ConcurrentDictionary + SemaphoreSlim
Per-clientId outbox (List<PendingFrame>) guarded by SemaphoreSlim(1,1). Retry uses Task.Delay with exponential backoff. ACK removes atomically.
C5

ConcurrentDictionary + Channel<Frame>
Maps clientId → Session. Each Session holds a Channel<Frame>(256, DropOldest). Reconnect resumes pending QoS 1 outbox from QoS Engine.
C6
Storage Engine
FileStream + PeriodicTimer + SemaphoreSlim
WAL append to FileStream. PeriodicTimer calls Flush(flushToDisk:true) every 2ms. SemaphoreSlim array acts as per-seq waiter — writers block until their seq is flushed.
C7

IConsensusLayer interface
Default: NoopConsensus (passthrough, zero overhead). Optional: DotNext.Raft (pure .NET Raft implementation, single NuGet). Toggle via appsettings.json.

Concurrency Model

Three domains, each with the minimum locking required. The rule: never hold a ReaderWriterLockSlim across an await. Acquire, read/write, release, then await.

StateStore — RWLS[64]
  • 64 shards, each ReaderWriterLockSlim(LockRecursionPolicy.NoRecursion)
  • Shard index: HashCode.Combine(topic) & 63 — avoids string hashing overhead
  • Read: EnterReadLock / ExitReadLock — many concurrent readers
  • Write: EnterWriteLock / ExitWriteLock — exclusive, milliseconds max
  • Lock held only for dictionary operation, never across I/O
QoS Outbox — SemaphoreSlim per client
  • SemaphoreSlim(1,1) per clientId outbox (async-safe mutex)
  • await outboxLock.WaitAsync(ct) before Enqueue/Ack/GetPending
  • outboxLock.Release() in finally block — guaranteed release
  • Retry loop holds no lock during Task.Delay — fully non-blocking
  • Registry of outboxes: ConcurrentDictionary<string, OutboxEntry>
Sessions — Channel<Frame> per conn
  • Channel.CreateBounded<Frame>(new BoundedChannelOptions(256) { FullMode = DropOldest })
  • Single reader (write loop task) — no lock needed on read side
  • TryWrite is lock-free via interlocked operations internally
  • Session registry: ConcurrentDictionary<string, Session>
  • Channel.Writer.Complete() signals write loop to exit cleanly
C# — Sharded StateStoreinternal sealed class Shard
{
    public readonly ReaderWriterLockSlim Lock = new(LockRecursionPolicy.NoRecursion);
    public readonly Dictionary<string, JsonElement> Data = new();
}

public sealed class StateStore
{
    private const int NumShards = 64;
    private readonly Shard[] _shards = Enumerable.Range(0, NumShards)
        .Select(_ => new Shard()).ToArray();
    private readonly IWalWriter _wal;

    private Shard Pick(string topic) =>
        _shards[(uint)HashCode.Combine(topic) % NumShards];

    public async ValueTask SetAsync(string topic, JsonElement payload, CancellationToken ct)
    {
        // 1. WAL first — durable before memory
        var seq = await _wal.AppendAsync(topic, payload, ct);
        await _wal.WaitFlushedAsync(seq, ct);  // blocks until 2ms fsync window

        // 2. Memory update — lock for minimum time, never across await
        var shard = Pick(topic);
        shard.Lock.EnterWriteLock();
        try   { shard.Data[topic] = payload; }
        finally { shard.Lock.ExitWriteLock(); }
    }

    public JsonElement? Get(string topic)
    {
        var shard = Pick(topic);
        shard.Lock.EnterReadLock();
        try   { return shard.Data.TryGetValue(topic, out var v) ? v : null; }
        finally { shard.Lock.ExitReadLock(); }
    }
}

// Session write channel — lock-free send path
public sealed record Session(
    string ClientId,
    Channel<Frame> SendChannel,
    CancellationTokenSource Cts
) {
    public bool TrySend(Frame f) => SendChannel.Writer.TryWrite(f);
}

public static Session CreateSession(string clientId) => new(
    clientId,
    Channel.CreateBounded<Frame>(new BoundedChannelOptions(256) {
        FullMode = BoundedChannelFullMode.DropOldest,  // QoS 0: drop on overflow
        SingleReader = true, SingleWriter = false
    }),
    new CancellationTokenSource()
);

Storage Engine

Five layers, same architecture as Go. The key .NET-specific detail: FileStream must be opened with FileOptions.Asynchronous for non-blocking writes, and Flush(flushToDisk: true) is the exact equivalent of Go's f.Sync().

WAL Writer
Async append of NDJSON entries. Monotonic sequence number via Interlocked.Increment.
Opens with FileMode.Append | FileOptions.Asynchronous | FileOptions.SequentialScan. Uses Utf8JsonWriter writing directly to a pooled ArrayBufferWriter<byte>, then RandomAccess.WriteAsync for the OS write. Single SemaphoreSlim serializes appends.
RandomAccess.WriteAsync · Interlocked.Increment · ArrayPool<byte>
Batched fsync
PeriodicTimer fires every 2ms. One Flush(flushToDisk:true) covers all pending appends.
Writers call await _flushGate.WaitAsync(ct) after Append. The flush loop calls _fileStream.Flush(flushToDisk: true), updates _flushedSeq, then calls _flushGate.Release(pendingCount) to wake all waiting writers. This is the exact Go sync.Cond.Broadcast() equivalent.
PeriodicTimer · SemaphoreSlim · FileStream.Flush(flushToDisk:true)
Segment Rotation
Rotate WAL file when it exceeds 64 MB. Sealed files are immutable.
Background task monitors new FileInfo(walPath).Length after each flush. On threshold: flush current, rename to wal-{seq:D8}.log, open new active file. FileSystemWatcher is not used — polling is simpler and avoids platform differences.
FileInfo.Length · File.Move · FileOptions.Asynchronous
Snapshots
Full StateStore dump every 5 min. Uses write-to-temp + File.Replace for atomic swap.
Iterate all 64 shards under read locks (held briefly per shard, not globally). Serialize to snapshot.tmp via Utf8JsonWriter. Call File.Replace(tmp, snapshot, backup) for atomic rename. On restart: deserialize snapshot → replay WAL from snapshot's last Seq.
File.Replace · Utf8JsonWriter · JsonSerializerContext (AOT-safe)
No-op Mode
NullWalWriter implements IWalWriter. All calls return immediately.
The IWalWriter abstraction means the StateStore doesn't know whether it's writing to disk or not. Inject NullWalWriter for pure in-memory operation. State is lost on restart. Suitable for ephemeral deployments and unit tests.
IWalWriter (interface) · NullWalWriter : IWalWriter
C# — WAL with Batched fsyncpublic sealed class WalWriter : IWalWriter, IAsyncDisposable
{
    private readonly FileStream _file;
    private readonly SemaphoreSlim _appendLock = new(1, 1);
    private readonly SemaphoreSlim _flushGate  = new(0);   // writers wait here
    private volatile long _seq;
    private volatile long _flushedSeq;
    private int _waitingWriters;

    public WalWriter(string path) =>
        _file = new FileStream(path,
            FileMode.Append, FileAccess.Write, FileShare.Read,
            bufferSize: 65536,
            FileOptions.Asynchronous | FileOptions.SequentialScan);

    public async ValueTask<long> AppendAsync(
        string topic, JsonElement payload, CancellationToken ct)
    {
        var buf = new ArrayBufferWriter<byte>();
        var writer = new Utf8JsonWriter(buf);
        var seq = Interlocked.Increment(ref _seq);
        writer.WriteStartObject();
        writer.WriteNumber("seq", seq);
        writer.WriteString("topic", topic);
        writer.WritePropertyName("payload");
        payload.WriteTo(writer);
        writer.WriteEndObject();
        await writer.FlushAsync(ct);
        buf.Write("\n"u8);  // NDJSON delimiter

        await _appendLock.WaitAsync(ct);
        try   { await _file.WriteAsync(buf.WrittenMemory, ct); }
        finally { _appendLock.Release(); }
        return seq;
    }

    public async ValueTask WaitFlushedAsync(long seq, CancellationToken ct)
    {
        if (Volatile.Read(ref _flushedSeq) >= seq) return;  // fast path
        Interlocked.Increment(ref _waitingWriters);
        await _flushGate.WaitAsync(ct);
    }

    // Called by background PeriodicTimer every 2ms
    public async Task FlushLoopAsync(CancellationToken ct)
    {
        using var timer = new PeriodicTimer(TimeSpan.FromMilliseconds(2));
        while (await timer.WaitForNextTickAsync(ct))
        {
            await _file.FlushAsync(ct);               // kernel buffer → disk
            Volatile.Write(ref _flushedSeq, _seq);
            var waiting = Interlocked.Exchange(ref _waitingWriters, 0);
            if (waiting > 0) _flushGate.Release(waiting);  // wake all blocked writers
        }
    }
}

NativeAOT Constraints AOT requires all types used in System.Text.Json serialization to be declared in a JsonSerializerContext with [JsonSerializable] attributes — no runtime reflection. This is a one-time setup (~30 lines) but is non-negotiable for AOT. Avoid any Activator.CreateInstance, dynamic types, or Assembly.Load in the broker core.

Go vs .NET — Key Differences Summary

AreaGo Version.NET VersionImpact
Concurrency model goroutine (M:N scheduler) Task + async/await (ThreadPool) Functionally equivalent. .NET ThreadPool auto-scales. async permeates all I/O.
Read-write locking sync.RWMutex ReaderWriterLockSlim RWLS is NOT async-safe. Must never hold across await. Use SemaphoreSlim(1,1) for async-guarded sections.
Typed channels chan T (select) Channel<T> (await ReadAsync) .NET channels have richer options: BoundedChannelFullMode, SingleReader hint, CompleteWriting for graceful shutdown.
Network I/O net.Conn + bufio.Reader System.IO.Pipelines.PipeReader Pipelines is zero-copy and handles partial reads automatically. More complex API but better throughput.
Binary size ~8 MB (static) ~15 MB (NativeAOT) / 60 MB (self-contained) NativeAOT nearly matches Go. Self-contained includes full runtime — significantly larger.
